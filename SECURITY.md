# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability within this project or any project derived from this template, please report it responsibly.

**Do not** open a public issue for security vulnerabilities.

Instead, please send a detailed report to the repository maintainer via private communication channels.

Include the following information in your report:

- A clear description of the vulnerability
- Steps to reproduce the issue
- Potential impact and severity assessment
- Any suggested mitigation or remediation steps

## Response Timeline

We will acknowledge receipt of your report within 72 hours and will provide an estimated timeline for addressing the issue.

## Supported Versions

Security updates will be applied to the latest version of this template.
For derived projects, maintainers are responsible for applying security fixes.

## Security Best Practices

When using this template:

- Keep dependencies up to date
- Never commit sensitive credentials or secrets
- Use environment variables for configuration
- Apply least privilege principles
- Enable security scanning tools in your CI/CD pipeline
